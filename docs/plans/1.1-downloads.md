## Purpose

- **🔷** Expose enough of WebKitGTK’s download surface that OLLMchat `Browser` can drop the Linux-only `#if` and run the same permission → place → progress → finish flow on Android.
- **🔷** Use Android System WebView hooks (`DownloadListener`, cookies) under the hood — not a parallel Soup-only path that bypasses the WebView session.
- **🔷** Keep permission and banner UX out of this repo.

## What Linux OLLMchat actually calls (contract to mirror)

From `Browser.vala` / **5.0.5** — this is the **required** consumer surface:

### Entry points

1. **Click / navigation → download**  
   Linux: unsupported MIME `decide_policy` → `response_decision.download()` → `NetworkSession.download_started`.  
   Android: **`DownloadListener.onDownloadStart`** (and any other path that means “this response is a file”) must produce the same **`download_started`** outcome.

2. **Tool `action: "download"`**  
   `web_view.download_uri(url)` → same `download_started` pipeline.

### Per-download object (`WebKit.Download`-shaped)

| Capability | Why OLLMchat needs it |
| ---------- | --------------------- |
| Request URI (`get_request().uri` or equivalent) | Permission key `browser_download#` + url; dedupe |
| Signal **`decide_destination(suggested_filename)`** → handler returns `true` and **does not** start I/O yet | Async `ChatPermission` before writing |
| **`set_destination(path)`** / **`set_allow_overwrite(true)`** | Start transfer only after Allow |
| **`cancel()`** | Deny permission; later **5.0.6** banner Cancel |
| Signal **`received_data`** + **`get_received_data_length()`** | `event.browser.download.progress` |
| Signals **`finished`** / **`failed(Error)`** | End event + tool `download()` await |

**Critical semantics (WebKit 2.40+ behaviour OLLMchat relies on):**

- **🔷** Returning `true` from `decide_destination` **without** `set_destination` means “I’ll decide later”.
- **🔷** Bytes must **not** hit disk until `set_destination` (or the download must be cancellable with no partial user-visible file).
- **🔷** Deny / error path must call **`cancel()`** (or fail) — never leave a hung download with neither destination nor cancel.

### Out of scope for this library (stay in OLLMchat)

- **🚫** `ChatPermission` / session allow keys  
- **🚫** `event.browser.download.*` / ActivityBanner  
- **🚫** Choosing Downloads folder vs home  
- **🚫** History / open / manager UI (**5.0.7**)

## Proposed public API (this repo)

Names mirror WebKitGTK where practical so `#if ANDROID` in `Browser` stays thin.

### `WebKitGtkAndroid.WebView`

- **🔷** `Download download_uri (string uri)` — start a download using the WebView’s cookie / session context; emit **`download_started`**.
- **🔷** signal `download_started (Download download)` on **`NetworkSession`** (same call site as Linux / webview2-gtk shape once session exists). Cookies already live there — do **not** put this on `WebView` alone.
- **✔️** `get_network_session()` + cookie manager already landed.

### `WebKitGtkAndroid.Download` (`GLib.Object`)

- **🔷** `string get_uri ()` (or request object with `.uri`)
- **🔷** signal `bool decide_destination (string? suggested_filename)`
- **🔷** `void set_destination (string destination_uri_or_path)`
- **🔷** `void set_allow_overwrite (bool allow)`
- **🔷** `void cancel ()`
- **🔷** `uint64 get_received_data_length ()`
- **🔷** signal `void received_data (uint64 data_length)` — WebKit passes chunk length; OLLMchat only reads total via `get_received_data_length()`
- **🔷** signal `void finished ()`
- **🔷** signal `void failed (GLib.Error error)`
- **💩** `string? mime_type` / `int64 content_length` — useful for progress_total (Linux currently sends `progress_total = 0`); expose if cheap from `DownloadListener`

### Host (Java / JNI)

- **🔷** `WebView.setDownloadListener` → JNI callback → Vala constructs `Download`, emits `download_started`.
- **🔷** Parse suggested filename from `Content-Disposition` (+ URL basename fallback) for `decide_destination`.
- **🔷** On `set_destination`: perform HTTP(S) GET with **`CookieManager.getCookie(url)`** (and User-Agent if available) so authenticated / session downloads match the page.
- **🔷** Progress + finish/fail callbacks onto the GTK/UI thread.
- **🔷** `cancel` aborts the in-flight transfer (DownloadManager remove / abort HttpURLConnection / OkHttp call — pick one in implement).

### Click / MIME path

- **🔷** `DownloadListener` covers the common “link / attachment / octet-stream” case — **required**.
- **💩** Full `decide_policy` / `ResponsePolicyDecision.download()` mirror — **only if** device testing shows Listener misses a case OLLMchat needs. Prefer not to invent policy API until proven.

## Android storage requirements

- **🔷** Caller (OLLMchat) still passes the destination string after permission (same as Linux `set_destination`).
- **🔷** Library must successfully write that destination **or** fail with a clear `GLib.Error`.
- **💩** Destination shape on Android (lock at implement / spike):
  1. Prefer a **real filesystem path** under app-accessible Downloads / cache if Pixiewood/`GLib.UserDirectory.DOWNLOAD` already resolves — keeps OLLMchat event `message` = path unchanged.
  2. If only MediaStore / `content://` works: either document that `set_destination` accepts `content://` / library-created display path, **or** add a small host helper `wka_host_downloads_dir()` that returns a writable path OLLMchat should use.
- **🚫** Do not show a system save picker from this library — destination is chosen by OLLMchat after permission.
- **🚫** Do not request storage permission UI here — OLLMchat / app manifest owns that.

---

## Cookies / session (non-negotiable)

- **🔷** Tool and click downloads must use the **same cookie jar** as the embedded WebView (Linux `download_uri` does).
- **🔷** Host fetch must attach `Cookie` header from `android.webkit.CookieManager` for the download URL.
- **✔️** Vala cookie **set/get** API is in (`NetworkSession` / `CookieManager`); downloads only need to **read** the same jar on transfer.

---

## Progress / cancel

- **🔷** Progress: report received bytes often enough for ActivityBanner pulse/fraction (even if `content_length` unknown → total 0, same as Linux today).
- **🔷** `cancel()` must stop I/O and emit **`failed`** or a cancelled error path so OLLMchat cleanup/`download()` await unblocks.
- **ℹ️** Banner Cancel button itself is OLLMchat **5.0.6**; this lib only needs a working `cancel()`.
