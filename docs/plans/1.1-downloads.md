**Split of responsibility (locked by 1.0 / 5.0.5):**

- **🔷** **This library** — WebKitGTK-shaped download *engine* API + actually transfer bytes to a destination the caller sets.
- **🔷** **OLLMchat** — permission UI (`ChatPermission`), activity events, `GLib.Notification`, in-flight dedupe, destination *policy* (which folder / filename).
- **🚫** No download-manager UI / history list in this library (**5.0.7** is OLLMchat).

---

## Purpose

- **🔷** Expose enough of WebKitGTK’s download surface that OLLMchat `Browser` can drop the Linux-only `#if` and run the same permission → place → progress → finish flow on Android.
- **🔷** Use Android System WebView hooks (`DownloadListener`, cookies) under the hood — not a parallel Soup-only path that bypasses the WebView session.
- **🔷** Keep permission and banner UX out of this repo.

---

## Shipped API (this repo)

### `WebKitGtkAndroid.WebView`

- **✔️** `Download download_uri (string uri)`
- **✔️** `get_network_session()` → cookies + **`download_started`**

### `WebKitGtkAndroid.NetworkSession`

- **✔️** signal `download_started (Download download)`
- **✔️** `get_cookie_manager()`

### `WebKitGtkAndroid.Download`

- **✔️** `get_request()` / `URIRequest.uri` / `get_uri()`
- **✔️** signal `bool decide_destination (string? suggested_filename)` (idle after started; return `true` to own destination)
- **✔️** `set_destination` / `set_allow_overwrite` / `cancel`
- **✔️** `get_received_data_length` + `received_data` / `finished` / `failed`
- **✔️** `get_mime_type` / `get_estimated_content_length` (from `DownloadListener` when present)

### Host

- **✔️** `DownloadListener` → JNI → Vala `download_started`
- **✔️** Suggested filename from `Content-Disposition` + URL basename
- **✔️** `HttpURLConnection` GET with `CookieManager` + User-Agent; progress / finish / fail on GTK idle
- **✔️** `cancel` disconnects in-flight transfer

### Destination

- **✔️** Caller passes a **filesystem path**; library writes there (mkdirs parent). Example uses `UserDirectory.DOWNLOAD` or tmp.
- **ℹ️** MediaStore / `content://` not required for v1 if app-writable Downloads/tmp works on device.
