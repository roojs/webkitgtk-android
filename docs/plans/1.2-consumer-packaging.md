## Purpose

- **рЯФЈ** Stop shipping only example executables вАФ ship an installable / linkable library so OLLMchat (and other apps) can `dependency('webkitgtk-android-1')`.
- **рЯФЈ** Mirror webview2-gtk consumer shape: pkg-config name + VAPI + shared lib; Android adds **Java host classes** in the same APK.
- **рЯФЈ** Keep a11y host entry points **off** `WebView` (same as Windows: host / facade, not WebKit-shaped widget methods).
- **рЯЪЂ** Do not wire OLLMchat `Browser` / tool registration here вАФ that is **5.0.2**.

## Locked decisions

### Meson product

- **рЯФЈ** Default Android configure (neither hello nor browser) builds the **library**.
- **рЯФЈ** `-Dandroid_browser=true` вЖТ library + browser example that **links** it.
- **рЯФЈ** `-Dandroid_hello=true` вЖТ hello exe only (**skip** library вАФ hello Pixiewood has no libsoup).
- **рЯФЈ** pkg-config / soname: **`webkitgtk-android-1`** / `libwebkitgtk-android-1.so`.
- **рЯФЈ** `meson.override_dependency('webkitgtk-android-1', вА¶)` so a Meson subproject works without a prefix install.

### VAPI (one file)

- **рЯФЈ** Single checked-in **`vapi/webkitgtk-android-1.vapi`** (like `webview2gtk-1.vapi`).
- **рЯФЈ** Contents: `WebKitGtkAndroid.*` widget API **plus** file-scope `wka_host_a11y_*` (ensure / walk / invoke / set_value / focus).
- **рЯЪЂ** No second `*-host.vapi`.
- **рЯЪЂ** No `a11y_*` methods on `WebView` вАФ Meson-generated library VAPI also must not grow those wrappers.
- **вДєпЄП** Meson still generates a build-tree VAPI from Vala sources for in-tree linking; keep the checked-in file aligned for prefix / wrap consumers.
- **вДєпЄП** Browser demo: `examples/browser/host-a11y.vala` redeclares host entry points locally (generated lib VAPI omits bare `extern`s).
- **вЬФпЄП** **Valadoc** on Linux: `meson setup build-docs -Ddocs=true && ninja -C build-docs docs/valadoc` (same pattern as webview2-gtk; no Android cross required).

### Java

- **рЯФЈ** Consumers must copy `lib/host/java/org/roojs/webkitgtk/android/*.java` into the app Java tree (same APK ClassLoader as the `.so`).
- **рЯФЈ** Helper: `scripts/android/install-webview-java.sh <java-src-root>`.
- **вЬФпЄП** Browser Pixiewood build already calls this.

### Pixiewood note

- **вДєпЄП** APK / `meson install` under Pixiewood often stages **runtime `.so` only** (devel `.pc` / headers / VAPI may not land in `.pixiewood/root`). Prefer **subproject + override_dependency** for compiling consumers.
